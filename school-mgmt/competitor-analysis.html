What "Plugin", "Partial" & "CBSE Only" Actually Means
🔌 Fedena — "Plugin" Items Explained Open-source platform — core is free, but many features require paid plugins installed separately by IT team
Feature Plugin Name Extra Cost? Install Difficulty What It Means in Practice
Biometric Integration Attendance Plugin Yes — Paid Medium School's IT team must install the plugin on self-hosted server, configure device drivers manually. No built-in device management panel.
GPS Tracking Transport Plugin Yes — Paid Hard Requires a third-party GPS hardware vendor integration. No live parent-tracking app view. School must purchase GPS hardware separately and configure API connections.
Online Payment Gateway Fee Payment Plugin Yes — Paid Medium Must self-integrate Razorpay / PayU credentials. No automatic reconciliation. Manual verification required for each online payment. No duplicate payment protection.
WhatsApp Integration Messaging Plugin Yes — Paid Hard Requires WhatsApp Business API approval from Meta (2–4 weeks), own WhatsApp Business Account, and plugin configuration by developer. Not ready out-of-the-box.
Push Notifications (FCM) Notification Plugin Yes — Paid Medium Requires Firebase project setup, app publishing to Play Store/App Store with school branding, and plugin integration. Not instant — takes weeks of setup.
SMS Alerts SMS Plugin Yes — Paid Easy Requires purchasing SMS credits from a gateway (Textlocal, Twilio). Plugin connects but school must manage credits and DLT registration separately.
Library Management Library Plugin Yes — Paid Easy Functional once installed. Basic book catalogue and issue/return. No barcode scanner integration without additional hardware setup. Fine management is manual.
Transport Management Transport Plugin Yes — Paid Easy Route and stop management available. No live GPS view for parents. Driver management is basic. No integration with fee module for transport fee billing.
Payroll Management HR Plugin Yes — Paid Medium Basic payroll calculation available. No Tally export. No ESI/PF automated filing. Indian tax compliance (TDS) must be calculated manually. No payslip email automation.
Hostel Management Hostel Plugin Yes — Paid Easy Room allocation and student assignment. Mess/food management not included. No integration with fee module for hostel billing automation.
Audit Trail Logging Plugin Free (basic) Easy Basic activity log available. Does NOT meet full IT Act 2000 requirements — no tamper-proof logging, no user-level change tracking per field, no OTP-verified data changes.

📋 "CBSE Only" — Chanakya ERP Board Support Limitation What changes when a school switches board type
Feature Area Chanakya ERP (CBSE Only) Chalo (CBSE + IGCSE + State) Impact on School
Report Card Format Pre-built CBSE format only (PT1 + HY + Annual). Fixed template. CBSE, IGCSE (Cambridge format), and custom State board templates. School selects format at setup. IGCSE / international schools cannot use Chanakya — report card format mismatch. Forced to do manual report generation outside the system.
Grade Scale CBSE A1–E scale only (33–100% range). Hard-coded. CBSE (A1–E), IGCSE (A*–G), or custom grade point scale configurable per school. IGCSE schools using A*–G grades cannot generate correct report cards. Grade conversion must be done manually in Excel.
Exam Structure CBSE structure: PT-1 (10) + NB (5) + Enrichment (5) + HY (80). Cannot change weightage. Configurable marks weightage per school, per term. Supports IGCSE coursework + exam split and State board structure. Schools with modified exam structure (e.g. 60+40 split, or project-based assessment) cannot configure Chanakya — forced to use default CBSE format only.
Subject Codes & Naming CBSE subject codes pre-loaded. Adding non-CBSE subjects is complex. Fully configurable subjects, codes, and department mapping. IGCSE subject codes supported. Schools offering IB, Cambridge or hybrid subjects face data entry issues — codes don't match, causing report card errors.
Affiliation Number Field CBSE affiliation number only on all documents Board affiliation number configurable — CBSE / CISCE / State board number Printed documents for ICSE / State board schools show wrong affiliation reference, causing compliance issues during inspections.
Switching Schools If a school switches from CBSE to State board, entire report card system breaks. Requires vendor customisation (paid). Board type can be changed in settings. Historical data retained. New format auto-applied from next academic year. Schools that plan to upgrade to CBSE or switch affiliations are locked into Chanakya's hard-coded template — migration costs are high.

Key Takeaways from Plugin / Partial / CBSE-Only Analysis

🔌

Fedena Plugins

10 of Fedena's key features are paid plugins requiring IT team installation, separate vendor contracts, and weeks of setup time. What looks like a ✅ on paper is actually a significant additional cost and technical burden for schools without IT staff.

⚠️

Partial Features

11 "Partial" items across competitors show features that exist in name but lack critical functionality — like Teachmint's "marks" having no CBSE report card, or Chanakya's "AI" being static bar charts. Partial features often still require manual Excel workarounds.

📋

CBSE Only (Chanakya)

Chanakya's CBSE-only limitation is a hard technical constraint — not a configuration option. Schools on IGCSE, State board, or those planning board migration are effectively excluded. Report cards, grade scales, and exam structures cannot be changed without paid vendor customisation.
